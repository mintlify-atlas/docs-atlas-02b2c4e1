## What is fast_opengl?

**fast_opengl** is a C++ library that wraps OpenGL 3.3+ with a simple, functional interface. Instead of managing VAOs, VBOs, shaders, and draw calls manually, fast_opengl provides high-level functions for common tasks:

- Render thousands of cubes or spheres with a single call using GPU instancing
- Load `.obj` models and place multiple instances with independent transforms
- Simulate N-body gravitational physics using leapfrog integration
- Load GLSL shaders from files at runtime
- Create an SDL2 window with an OpenGL 4.0 core context in one call

## Key features

<CardGroup cols={2}>
  <Card title="Instanced rendering" icon="layer-group">
    Render up to millions of cubes or icospheres in a single draw call using `glDrawElementsInstanced`. Each instance has its own position, scale, rotation, and texture index.
  </Card>
  <Card title="OBJ model loading" icon="box-open">
    Load any `.obj` file via tinyobjloader. Multiple models and instances are managed in a central scene system with lazy matrix updates.
  </Card>
  <Card title="N-body physics" icon="atom">
    Attach dynamic bodies to rendered objects and simulate gravitational attraction between all bodies using a Leapfrog (Verlet) integrator with softening.
  </Card>
  <Card title="Runtime GLSL shaders" icon="file-code">
    Shaders are read from `.glsl` files at runtime, making it easy to iterate without recompiling. Each subsystem compiles and links its own shader program.
  </Card>
  <Card title="Texture arrays" icon="images">
    Bulk-load PNG textures into a `GL_TEXTURE_2D_ARRAY` using SDL2_image. Each instance references a texture by index, enabling varied appearances with one draw call.
  </Card>
  <Card title="Procedural icospheres" icon="globe">
    Generate icospheres at any subdivision level. Higher divisions produce smoother spheres at the cost of more vertices.
  </Card>
</CardGroup>

## Architecture overview

fast_opengl is organized into independent subsystems, each with its own header and source file:

| Module | Header | Purpose |
|---|---|---|
| Window | `createWindow.h` | SDL2 window + OpenGL context creation |
| Shapes | `shapes.h` | Instanced cubes, spheres, icospheres, grid |
| OBJ Rendering | `obj.h` | OBJ model loading, instancing, transforms |
| Physics | `phy.h` | N-body gravity simulation |
| GLSL Loader | `glslread.h` | Read shader source files from disk |

Each subsystem maintains its own internal state via static globals and exposes a small, flat C-style function API callable from any C++ translation unit.
